Best path forward

A simple, local-first roadmap for the open IPTV app work. The bias is deliberate: stabilize the single active provider/session model, make existing Movies/Series/Search/Playback honest, then add persisted user state. Defer profiles, downloads, EPG, and indexing until the prerequisites are real.

Foundation Library UX Playback + Watch Favorites + For You Live TV Profiles + Downloads

Priority order

  1. P0 Foundation: lock provider/session/catalog invariants, type-aware catalog identity, lazy hydration semantics, and DB-backed prefix visibility. Open plan
  2. P1-A Library honesty: fix browse/search loading vs empty states, partial local coverage messaging, shared query normalization, and keep search local. Open plan
  3. P0/P1 Playback + watch activity: persist resumable progress, clean advanced controls, keep preferences device-global until profiles exist. Open plan
  4. P1 Discovery: add persisted favorites and a deterministic local For You screen from favorites/watch/catalog rows. Open plan
  5. P1-B/P1-C Library enrichment: add minimal nullable metadata and series episode persistence only after the core UX is honest. Jump to section
  6. P1/P2 Live TV: basic channels first using the same local catalog/player pattern; EPG/catch-up later. Open plan
  7. P2 Deferred complexity: profiles before downloads, downloads after profile-aware user state and a playback-source-origin seam. Open plan
