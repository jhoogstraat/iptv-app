P1 · Current surfaces first

Library browsing, search, filters, and details

Make the library surfaces already in the app truthful and complete before introducing a search index, recommendation system, downloads, or profiles.

Recommended second

Decision

Use the existing SQLiteData/GRDB + SwiftUI query path. Do not add SearchIndexStore, SearchOrchestrator, full text search, remote search calls from screens, or relevance-ranking infrastructure until measured local filtering fails acceptance.

Phased plan

PhaseWorkAcceptance
P1-AMake Browse/Search states honest: unhydrated, loading, empty, populated, and failed. Keep category hydration local and visible.No screen renders skeleton tiles as a false empty state. Search explains partial local coverage when applicable.
P1-AUnify query normalization and shared filters/sorts across BrowseScreen, SearchScreen, and LibraryFilterEngine.Scoped and global search return consistent results for the same local rows and active filters.
P1-ARoute browse/search/media rails to real detail screens and explicit unsupported states.Movie/series rows never navigate to generic placeholder content when a local detail route exists.
P1-BAdd nullable metadata fields only when the Xtream DTOs actually provide them: synopsis, year, runtime, cast, genre/language if reliable.Details show real values or explicit unavailable copy; filters are only enabled for populated local fields.
P1-CPersist series seasons/episodes as local rows linked to parent series.Series detail can list playable episodes and player source resolution handles episode rows.

Do not do yet

  • Do not build a parallel indexing subsystem from the older search specs unless the direct local query path cannot meet performance targets.
  • Do not expose filters for genre, year, language, audio language, subtitle language, or added-date until those fields are actually populated locally.
  • Do not remote-fetch from view bodies to fill routine screen content. Remote calls are sync/detail enrichment/playback resolution only.
  • Do not implement favorites, downloads, continue watching, or recommendation rows in this workstream; link to their plans instead.

How to handle the next workstream

After P1-A is complete, implement Playback and watch activity. That creates the smallest durable user-state foundation needed for Continue Watching, For You, and honest detail/player controls.

Reviewer notes

The high-risk mistake is reviving obsolete search-index architecture from older specs. Keep this page as the source of truth: direct local queries first, index only after measured need.
