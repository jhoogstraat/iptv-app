Favorites and For You

Build the smallest persisted user-state layer that makes Favorites, detail favorite controls, Continue Watching, and a non-placeholder For You page real.

Discovery without magic

Decision

Add provider-scoped favorites and consume provider-scoped watch activity. For You v1 should be deterministic and local: Continue Watching, Favorites, top-rated catalog rows, recently updated rows, and category-prefix visibility. Do not create remote recommendation calls, a recommendation index, profiles, downloads, or full-text search as part of this work.

Implementation phases

PhaseWorkAcceptance
1Add provider-scoped favorites rows keyed by media type + remote source ID with snapshot title/artwork/category fields.Favorite state persists across app restart and is isolated by active provider reset semantics.
2Wire Favorite controls in detail/player surfaces to the store.Favorite/unfavorite updates all visible surfaces without app restart.
3Replace FavoritesScreen placeholder with local query sections and empty/error states.Favorites tab lists favorite movies/series with active filters/grouping where useful.
4Replace ForYouScreen placeholder with deterministic local rails.For You shows Continue Watching when watch activity exists, a hero from recent/top-rated local rows, Favorites rail when populated, and no fake recommendations.
5Cross-surface discoverability.Search/detail/browse/player reflect favorite and progress state consistently, using shared content keys.

For You rail design

Continue Watching

From watch_activity, unfinished only, sorted by last watched.

Favorites

From persisted favorites, joined to live local Media rows when available.

Top Rated

Local catalog rows with non-nil rating; separate Movies and Series rails.

Recently Updated

Local catalog rows by updatedAt, respecting hidden category prefixes.

Spec cross-links

  • Favorites: owns favorite persistence and UX.
  • For You: owns landing/discovery behavior.
  • Media Details: owns Favorite/Resume actions on detail pages.
  • Search: can expose favorites/continue-watching once persisted.
  • Playback plan: supplies watch activity.
  • Deferred complexity plan: profiles later widen user-state keys.

How to handle the next workstream

After Favorites and For You are real, move to either series episode/detail enrichment or basic Live TV. Choose episode enrichment first if playback resume for series is the product priority; choose Live first if channel browsing is the priority. Do not start downloads before profiles/source-origin prerequisites.

Reviewer notes

The important constraint is avoiding fake personalization. Every rail must be explainable from persisted local rows available today.
